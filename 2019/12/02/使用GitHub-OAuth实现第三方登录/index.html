使用GitHub_OAuth实现第三方登录

发表于 2019-12-02

归纳这里的知识点就是：第三方登录功能。

我选择的平台是GitHub

OAuth 2.0

​ OAuth（开放授权）是一个开放标准，允许用户让第三方应用访问该用户设置的内容而无需将用户名和密码提供给第三方应用。

​ 这个协议在 认证 和 授权 的过程中涉及到：

  • 服务提供方（如GitHub，其上存储了用户登录名，Email等信息）
  • 用户
  • 客户端（例如自己的博客，需要服务方向我的博客提供用户的基本信息）

​ OAuth协议的 认证 和 授权 的过程：

  • 用户访问我的博客后，我的博客请求GitHub获取该用户的信息
  • 跳转到GitHub授权页面后，用户同意我获取他的信息
  • 博客活得GitHub提供的 授权码 ，使用该授权码向GitHub申请一个 令牌
  • GitHub对博客提供的授权码进行 验证 ，无误后，发放一个令牌给博客端
  • 博客使用令牌，向GitHub获取用户信息
  • GitHub确认令牌无误，返回给博客基本信息。

使用GitHub的OAuth服务的具体过程

  • Settings > Developer setting > OAuth application

各项配置查看官方文档

​ 使用第三方网站登录的流程如下：

  • 跳转到GitHub用户授权页面（client_id必须传，其他参数有需要可以传）

  • 用户同意授权后，链接地址会跳转到配置页面内的Authorization callback URL填写的地址。并且会带上一个code参数，这个参数在后面获取用户的token是必须的一个参数。获取之后，客户端会将这个code传到服务器后台，后台调用这个API，传入client_id，client_secret, code这三个参数，客户端可以获取到一个access_token

  • 之后再获取这个API，就可以获取到基本的用户信息了。 用户的基本信息内容如下所示， 根据第一步传入的不同的 scope，获取到的用户信息也是不同的。博客后台使用 login 字段作为用户的唯一标示 。
