我的博客搭建过程

发表于 2019-12-02

准备条件

  • GitHub提供了 GitHub Pages ：帮助架设一个静态网站 （解决没有服务器的问题）
  • Hexo博客框架：轻量级、MarkDown直接写文章、直接将文章编译成静态网页文件并发布（文章相关信息不必存储在数据库中了，属于纯静态页面，解决没有数据库的问题）
  • GitHub Pages允许每个账户创建一个名为{username}.github.io的仓库、并自动为这个仓库分配一个github.io的二级域名（解决了没有域名的问题）、也可以支持自定义域名

新建项目

  • 在GitHub新建一个仓库，名称为{username}.github.io

  • 为了和后面的配置相呼应，应该去配置GitHub的SSH链接

  • Hexo

gitalk

由于Hexo是静态博客，而且没有连接数据库的功能，所以它的评论功能不可自行集成。但可以集成第三方的服务。

Next主题提供了多种评论插件的集成：changyan | disqus | disqusjs | facebook_comments_plugin | gitalk | livere | valine | vkontakte

这里选择以gitalk为例 ，它是利用GitHub的issue当评论的。

  • 首先需要在GitHub注册一个OAuth Application。注册完拿到Client ID 和 Client Secret就行了。

  • 在comments区域配置使用gitalk
